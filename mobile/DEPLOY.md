# Deploying Kitcho Family Mobile App to Google Play Store

## Prerequisites

1. Google Play Developer Account
   - Sign up at [Google Play Console](https://play.google.com/console)
   - Pay the one-time registration fee ($25)

2. App Assets
   - App icon (512x512 PNG)
   - Feature graphic (1024x500 PNG)
   - Screenshots for different devices (phone, tablet)
   - Privacy policy URL
   - App description and other store listing details

## Build Configuration

The app is already configured with the necessary Android settings in `app.config.js`:
- Package name: com.kitchofamily.mobile
- Version code: 1
- Required permissions: INTERNET

## Building the Release APK

1. Install EAS CLI globally:
```bash
npm install -g eas-cli
```

2. Log in to your Expo account:
```bash
eas login
```

3. Configure EAS Build by running:
```bash
eas build:configure
```

4. Create a build profile in `eas.json`:
```json
{
  "build": {
    "preview": {
      "android": {
        "buildType": "apk"
      }
    },
    "production": {
      "android": {
        "buildType": "app-bundle"
      }
    }
  }
}
```

5. Generate a keystore for signing:
```bash
eas build:configure --platform android
```

6. Build the production bundle:
```bash
eas build --platform android --profile production
```

## Submitting to Google Play Store

1. Access Google Play Console
   - Log in to [Google Play Console](https://play.google.com/console)
   - Create a new application

2. Store Listing
   - Upload all required graphics
   - Fill in app description
   - Provide privacy policy URL
   - Add app category and content rating

3. Release Management
   - Upload the AAB (Android App Bundle) generated by EAS
   - Create a release
   - Complete the rollout information
   - Submit for review

## App Signing

EAS Build handles app signing automatically. The signing credentials are stored securely in your EAS project.

## Important Notes

1. Testing
   - Test the release build thoroughly before submission
   - Ensure all API endpoints are configured correctly for production
   - Verify all features work as expected

2. Updates
   - Increment the `versionCode` in `app.config.js` for each new release
   - Keep track of changes for release notes

3. Compliance
   - Ensure the app complies with Google Play Store policies
   - Maintain an up-to-date privacy policy
   - Handle user data according to GDPR and other relevant regulations
